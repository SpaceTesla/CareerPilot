# Feature Specification: Intelligence Agent (F3.4)

## 1. Purpose
The Intelligence Agent is a central synthesis node in the CareerPilot multi-agent graph runtime. It replaces simple heuristic-based scoring engines with a generative reasoning agent that evaluates a user's Career Health Score, maps their Position Delta against target positions, aggregates Opportunity Intelligence rankings, and conducts compensation benchmarking. The Intelligence Agent translates raw quantitative scores into a cohesive, written "Intelligence Report" that provides deep strategic explanations and maps actionable improvement tasks (e.g., adding a specific skill, modifying experience positioning) backed by concrete evidence.

---

## 2. User Value
The Intelligence Agent powers the "Explainability First" core philosophy of CareerPilot. Users never receive career health scores, opportunity fits, or gap analyses without a detailed, transparent breakdown of the "why." By presenting evidence-backed positioning strategies (e.g., "Your fit score is 68% because this role requires Kubernetes orchestration, which is missing from your active profile but mentioned in your historical project logs"), the agent builds user trust and gives them a clear blueprint to optimize their career progression.

---

## 3. Requirements
* **Heuristic Integration**: Ingest calculations from Career Health Score Engine V1 (F1.7), Position Delta Engine (F1.8), and Opportunity Intelligence (F2.7).
* **Explanation Synthesis**: Generate structured explanations summarizing why a user fits a role, where the gaps lie, and how they rank in the market.
* **Evidence Attribution**: Intersect profile blocks, resumes, and market postings. Tag every recommendation with specific cross-references (e.g., "Postings research indicates JAX is prioritized in 73% of ML roles at Stripe").
* **Actionable Improvement Roadmap**: Generate a prioritized sequence of modifications the user should make to their profile, resume, or skills list to close the delta.
* **Compensation Benchmarking Synthesis**: Correlate parsed salary bands with user salary targets, providing advice on negotiation leverage and market percentiles.
* **Intelligence APIs**: Expose endpoints to request reports, review specific explanations, and check improvement roadmaps.
* **Evaluation Suite**: Implement standard LLM-as-a-judge tests to evaluate readability, alignment confidence, and attribution truthfulness.

---

## 4. Database Changes
No standalone database tables are created exclusively for this agent, as all compiled outputs are persisted inside the LangGraph state checkpoints (`agent_checkpoints` via LangGraph Foundation F3.1) and standard outcome records. However, we log intelligence summaries to historical audit collections for downstream calibration.

### PostgreSQL Tables

#### `agent_intelligence_reports`
Caches the structured outputs generated by the Intelligence Agent for quick dashboard queries.
```sql
CREATE TABLE agent_intelligence_reports (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    thread_id VARCHAR(255) NOT NULL REFERENCES agent_sessions(thread_id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    overall_health_score NUMERIC(5, 2) NOT NULL,
    position_delta_score NUMERIC(5, 2) NOT NULL,
    fit_score NUMERIC(5, 2) NOT NULL,
    structured_explanation JSONB NOT NULL, -- contains evidence, roadmap, and salary benchmarking
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP NOT NULL
);

CREATE INDEX idx_intelligence_reports_user_id ON agent_intelligence_reports(user_id);
CREATE INDEX idx_intelligence_reports_thread ON agent_intelligence_reports(thread_id);
```

## 6. Domain Models

```python
from pydantic import BaseModel, Field
from typing import List, Dict, Any, Optional
from uuid import UUID
from datetime import datetime

class EvidenceBlock(BaseModel):
    claim: str
    source_reference: str = Field(description="The specific ID, posting, or document source backing this claim.")
    source_type: str = Field(description="Type of source: e.g. 'job_post', 'peer_benchmark', 'resume_experience'")
    confidence_rating: str = Field(description="E.g., 'high', 'medium', 'low'")

class RoadmapItem(BaseModel):
    priority_order: int
    actionable_task: str = Field(description="Concrete step the user must take (e.g., 'Add Python design patterns under projects').")
    estimated_impact: str = Field(description="Expected fit improvement or career score impact.")
    associated_gaps: List[str] = Field(description="Which gaps this task resolves.")

class CompensationAnalysis(BaseModel):
    market_percentile: float
    salary_range_min: int
    salary_range_max: int
    negotiation_advice: str

class IntelligenceReportPayload(BaseModel):
    overall_health_score: float = Field(ge=0.0, le=100.0)
    position_delta_score: float = Field(ge=0.0, le=100.0)
    fit_score: float = Field(ge=0.0, le=100.0)
    summary_explanation: str
    evidence_trail: List[EvidenceBlock]
    profile_roadmap: List[RoadmapItem]
    compensation_context: CompensationAnalysis
```

## 7. Services

### `IntelligenceAgentService`
* **Method**: `generate_intelligence_report(thread_id: str, state: CareerPilotState) -> IntelligenceReportPayload`
  * Executes the main intelligence synthesis node. Combines the user profile with the outputs of F1.7, F1.8, and F2.7, constructs structured LLM prompts, generates evidence associations, compiles the profile improvement roadmap, and returns the unified payload.
* **Method**: `persist_intelligence_report(thread_id: str, report: IntelligenceReportPayload) -> UUID`
  * Serializes and stores the generated report in `agent_intelligence_reports` for fast dashboard retrieves.

## 9. Background Jobs
None. The Intelligence Agent runs synchronously within the LangGraph orchestrator thread context triggered by the Supervisor.

---

## 10. Acceptance Criteria
* **AC 1**: Given completed scoring runs for health score, delta, and opportunity fit, when the Intelligence Agent runs, it must produce an `IntelligenceReportPayload` containing a structured roadmap with prioritized actions.
* **AC 2**: Given any claim generated in the summary explanation, the report must contain at least one matching `EvidenceBlock` attributing the claim to a source.
* **AC 3**: The agent must output valid JSON conforming exactly to the Pydantic schema structure.

---

## 11. Edge Cases
* **Missing Downstream Score Data**: If the Position Delta score or Opportunity score failed to calculate in preceding graph steps, the Intelligence Agent must dynamically trigger heuristic calculations locally to prevent generating empty report blocks.
* **Conflicting Evidence**: If a company's job posting lists "No remote work" but historical memory says "Hires remote workers", the agent must log the conflict and synthesize a balanced warning explanation (e.g., "Official policy is onsite, though peer cohort outcomes indicate remote exception opportunities exist").
* **LLM Output Failure**: If the LLM generates a text output that fails JSON schema parsing after 3 retries, the agent must fall back to a deterministic backup compiler that structures the raw text into template blocks.

---

## 12. Test Requirements
* **Unit Testing**:
  * Test Pydantic model validations against invalid ranges (e.g., negative scores, health scores > 100.0).
  * Assert evidence mapping functions correctly pair claims with source files.
* **Integration Testing**:
  * Feed mock outputs from F1.7, F1.8, and F2.7 into `IntelligenceAgentService` and assert database persistence in `agent_intelligence_reports`.
* **Agent/Workflow Evaluation**:
  * Evaluate synthetic outputs against a test dataset of 30 mock profiles. A human or evaluator LLM must score the attribution truthfulness at >= 95% (zero hallucinated sources).
